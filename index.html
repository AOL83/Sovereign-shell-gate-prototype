Sovereign owner function v0.1

Owner key.
Not password soup.

This prototype creates a virtual owner card, stores its private signing key in the browser’s local key store, and uses an emulated reader to perform a cryptographic challenge-response unlock. The shell never compares a plain password. The card proves ownership by signing a fresh challenge. Look at us, making the welcome mat slightly less embarrassing.

Virtual CardHolds a non-extractable ECDSA private key in IndexedDB.
ReaderCreates a random challenge and asks the card to sign it.
VerifierChecks signature with the stored public key before unlocking.
This is a local browser prototype, not final production security. It proves the owner-key mechanism. Real production owner control should verify challenges server-side or via WebAuthn/passkeys. Browser-only authority can still be bypassed by someone controlling the browser. Yes, the goblin is still in the room.
